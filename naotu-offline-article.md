# 百度脑图下线了，我让claw撸了个能私有部署的替代品

百度脑图（naotu.baidu.com）正式停止服务，不得不导出之前的脑图文件……

对很多人来说，这是一个用了好几年的工具——免费、即开即用、没有广告、没有额度限制。但它正式下线了。

今天我让 claw 撸了一个能私有部署的替代品，叫 **my-minder**。

---

## 它是什么

一个在线思维导图工具，界面和百度脑图差不多，但数据存在你自己的 GitHub 私有仓库里。

你的脑图，就是 GitHub 仓库里的一个个文件。你能看到每一次修改记录，能 clone 下来，能随时迁走。没有任何第三方平台能把你的数据"下线"掉。

---

## 核心功能

**编辑器本身**，基于百度开源的 KityMinder 内核，支持节点优先级（P1–P9）、进度标记、超链接、图片、备注，用起来和百度脑图基本一致。

**文件管理**，左侧是文件树，支持新建、重命名、删除、移动，还能新建文件夹分类管理。文件列表会显示每个文件在 GitHub 上的最近修改时间，一眼看出哪些是最近在用的。

**XMind 互通**，这是我觉得最实用的一点。支持直接导入 `.xmind` 文件（XMind 8 和 XMind 2020+ 格式都支持），超链接、备注、标签、优先级、进度这些信息都能完整保留，不会丢。编辑完也可以导出回 `.xmind`，和 XMind 客户端双向流转。

**GitHub 登录**，用 OAuth 授权，不需要注册新账号，不需要记密码。脑图文件全都保存在你自己的 GitHub 选择的仓库内。

---

## 私有化：数据完全自己掌控

这是和百度脑图、幕布、ProcessOn 这类产品最本质的区别。

你的数据存在自己的 GitHub 仓库，格式是标准 JSON，可读、可迁移、有版本历史。自己的脑图文件，完全自己掌控。

---

## 怎么部署

Fork 仓库 → 在 GitHub 创建一个 OAuth App → 部署到 Vercel，填三个环境变量，完成。全程大概 10 分钟，不需要服务器，Vercel 免费套餐够用。

部署完访问你自己的域名，用 GitHub 账号登录，选一个仓库作为存储，就可以开始用了。

---

## 技术栈备注

Vue 2 + Vuex，编辑器用的 vue-minder-editor-extended，部署在 Vercel（前端静态 + Serverless Function 处理 OAuth），XMind 转换用的是我自己写的 [@ljheee/xmind-parser](https://www.npmjs.com/package/@ljheee/xmind-parser)，已发布到 npm，支持浏览器端直接运行。
